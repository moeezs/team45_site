Development Plan

Project Name: Bounce

Client Name: Krishh Lala

Team Members: Abdul Moeez Shaikh, Daryl John, Grady Rueffer, Gagan Bhattarai

Purpose:

  • Krishh Lala is a passionate basketball fan who wants a deeper way to engage with the sport beyond watching games. Additionally, he sees this game as a valuable tool for basketball enthusiasts to analyze team compositions and player efficiency in a strategic setting. Also, he wants to use the simulation for his parlays, simply to see what outcomes are more likely to happen, so that he can make more informed decisions.
  • Bounce is a text-based basketball simulation game where users build custom teams from a database of real player stats. They compete against pre-built teams like the Detroit Pistons in simulated matches, making strategic decisions such as passing, shooting, and substituting players. The game uses real-life stats to determine player performance, ensuring a realistic and immersive experience.

Data Description:

The game will require the following database tables:

1. Players Table (Stores player statistics and attributes)

  • player_id (INT, PRIMARY KEY)
  • name (VARCHAR)
  • position (VARCHAR)
  • team (VARCHAR)
  • points_per_game (FLOAT)
  • assists_per_game (FLOAT)
  • rebounds_per_game (FLOAT)
  • steals_per_game (FLOAT)
  • blocks_per_game (FLOAT)

2. User Team Table (Stores user-created teams)

  • team_id (INT, PRIMARY KEY)
  • user_id (INT, FOREIGN KEY referencing Users)
  • team_name (VARCHAR)

3. User Team Players Table (Associates players with user teams)

  • id (INT, PRIMARY KEY)
  • team_id (INT, FOREIGN KEY referencing User Team)
  • player_id (INT, FOREIGN KEY referencing Players)

4. Matches Table (Stores match results and statistics)

  • match_id (INT, PRIMARY KEY)
  • user_team_id (INT, FOREIGN KEY referencing User Team)
  • opponent_team (VARCHAR)
  • result (VARCHAR)
  • score (VARCHAR)

Functions Performed by the App:

  • User selects players to create a custom team (INSERT into User Team and User Team Players tables)
  • Game simulates match using real player statistics(SELECT player stats, calculate outcomes, INSERT match results)
  • User makes strategic decisions during gameplay(UPDATE game state based on user inputs)
  • User views match history and statistics(SELECT from Matches table)
  • User wants to make a new team(DELETE old User Team and INSERT into User Team and User Team Players tables)

Role of PHP and JavaScript:

PHP:

  • Handles database interactions (INSERT SELECT etc. operations on teams, players, matches)
  • Processes game simulations on the server
  • Manages user sessions and authentication

JavaScript:

  • Provides interactive user interface (team selection, in-game decisions)
  • Sends user actions (pass, shoot, etc.) to the backend for processing
  • Displays match progress and outcomes dynamically

Roles of Team Members:

  • Abdul Moeez Shaikh: Frontend development (JavaScript, UI implementation)
  • Daryl John: Game logic and simulation algorithm
  • Grady Rueffer: Backend development (PHP, database management)
  • Gagan Bhattarai: Project Management, JavaScript, testing, and documentation

The Increments:

Increment 1 (Week 11):

  • Database setup and fetching player data
  • Basic team creation functionality

Increment 2 (Week 12.1):

  • Game logic implementation (stat-based match simulation)
  • Basic UI for match interaction

Increment 3 (Week 12.2):

  • Polishing UI and game flow
  • Final testing and debugging
  • Deployment and presentation to client

Client Feedback:

After presenting the initial idea to Krishh Lala, he was excited about the ability to simulate real basketball games using actual player stats. He suggested:

  • Adding a difficulty setting to balance the gameplay.
  • Allowing users to save and load their teams for replayability.
  • Ensuring that player fatigue and stamina affect gameplay to increase realism. (Might be too complicated to implement, so we will leave it out for now)
