Development Plan

Project Name: Hoops Dynasty (Changed from Bounce)

Client Name: Krishh Lala

Team Members: Abdul Moeez Shaikh, Daryl John, Grady Rueffer, Gagan Bhattarai

Purpose:

  • Krishh Lala is a passionate basketball fan who wants a deeper way to engage with the sport beyond watching games. Additionally, he sees this idea as a valuable tool for basketball enthusiasts to analyze team compositions and player efficiency in a strategic setting. Also, he wants to use the simulation for his parlays, simply to see what outcomes are more likely to happen, so that he can make more informed decisions.
  • Hoops Dynasty is a choice-based basketball simulation game where users build custom teams from a database of real player stats. They compete against teams made by other users in simulated matches, making strategic decisions such as passing, shooting, and defending. The game uses real-life NBA stats to determine player performance, ensuring a realistic and immersive experience for our users.

Data Description:

The game will require the following database tables:

hoopsdynastyplayers Table (Stores all player stats and attributes) (Changed from Players. Also changed the stats that we use in the table fields)

  • player_id (INT, PRIMARY KEY)
  • player_name (VARCHAR)
  • player_team (VARCHAR)
  • three_point_percentage (INT)
  • two_point_percentage (INT)
  • free_throw_percentage (INT)
  • blocks_per_game (INT)
  • steals_per_game (INT)
  • personal_fouls_per_game (INT)
  • photo (LONGBLOB)

hoopsdynastyteams Table (Stores all user-created teams) (Changed from User Team. Also changed the table functionality to store full teams created by users as one entry)

  • id (INT, PRIMARY KEY)
  • creator (VARCHAR)
  • team_name (VARCHAR)
  • access (INT)
  • center (VARCHAR)
  • power_forward (VARCHAR)
  • small_forward (VARCHAR)
  • point_guard (VARCHAR)
  • shooting_guard (VARCHAR)

hoopsdynastyusers Table (Stores all account login info)

  • id (INT, PRIMARY KEY)
  • username (VARCHAR)
  • password (VARCHAR)
  • firstname (VARCHAR)
  • lastname (VARCHAR)
  • accesslevel (INT)
  • email (VARCHAR)

Functions Performed by the App:

(Made some changes to how the tables get used in our implementation)

  • User logins in/creates a new account (INSERT/SELECT from the hoopsdynastyusers table)
  • User selects players to create a custom team, or selects a previously made team (INSERT/SELECT teams from the hoopsdynastyteams table)
  • User plays against teams made by other users (SELECT teams from the hoopsdynastyteams table)
  • Game simulates match using real player statistics (SELECT player stats from the hoopsdynastyplayers table and calculates the outcome)
  • User makes strategic decisions during gameplay (Game logic is done through Javascript/PHP)

Role of PHP and JavaScript:

PHP:

  • Handles database interactions (INSERT SELECT etc. operations on the database tables)
  • Processes game simulations on the server
  • Manages user sessions and authentication

JavaScript:

  • Provides interactive user interface (team selection, in-game decisions)
  • Sends user actions (pass, shoot, etc.) to the backend for processing
  • Displays match progress and outcomes dynamically

One of the biggest challenges I faced was getting the logic of the quicktime events that happen randomly to feel smooth and realistic, while still being simple enough for users to follow. I also worked on making the game log show a detailed description of events, which would help users understand how their choices impacted the game's outcome. Fine-tuning the balance between realism and fun gameplay took several attempts, but I enjoyed the process and learned a lot about connecting PHP with JavaScript and making the AJAX interactions feel seamless.

  • Grady Rueffer: Designed and implemented all of the three main tables, as well as the scripts and frontend needed for the player selection and team creation. (PHP, Javascript, Database Management)

    As the backend developer, I was responsible for the database, queries, and DOM interaction with database content. You'll find most of my work in the teambuilder, as it is the most backend intensive in establishing player created teams, querying for player data, and implementing search behaviour.
